Welcome to the Hello OnCall Functions

01- Callable function precisa se chamanda no front-end com o firebase callable function. Callable não se retorna o Data, ao contrario do Request Functions onde retornamos o RESPONSE Neste caso, o retorno é feito pelo firebase, e não pelo código. Callable function não precisa de cors, pois o firebase já faz isso para nós.

02- Em Callable Functions, você não retorna um Response HTTP diretamente (como em onRequest). Em vez disso, você retorna um objeto/dados, e o Firebase SDK encapsula isso em "{{ '{' }} data: ... {{ '}' }}" automaticamente. No front-end, você acessa os dados via result.data (como no exemplo acima). Em funções HTTP tradicionais (onRequest), você precisa gerenciar o Response manualmente (incluindo CORS).

03- Neste caso, o retorno é feito pelo firebase, e não pelo código. O Firebase SDK lida com a serialização/deserialização dos dados e o formato da resposta. Você só precisa retornar um objeto/dados na sua função Cloud.

04- Callable function não precisa de CORS, pois o firebase já faz isso para nós. Callable Functions não exigem configuração manual de CORS, pois o Firebase SDK gerencia a comunicação por meio de seu próprio protocolo (não é uma requisição HTTP direta). Já em funções HTTP tradicionais (onRequest), você precisa configurar CORS manualmente.
